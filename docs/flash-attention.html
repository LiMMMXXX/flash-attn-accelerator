The Memory Wall

Attention is compute-bound — or is it?

On modern GPUs, the bottleneck isn't floating-point math. It's the time spent reading and writing intermediate tensors to GPU memory — especially that N×N attention matrix.

HBM · High Bandwidth Memory

GPU DRAM — the slow giant

Where the large N×N attention matrix S = QKT lives. HBM is big (40–80 GB) but reading and writing it runs at just 1.5–2 TB/s — the bottleneck for every standard attention implementation.

1.5 TB/s bandwidth
SRAM · Static Random-Access Memory

On-chip cache — the fast sliver

Each GPU streaming multiprocessor has a small pool of SRAM (~100 KB) that runs at 19 TB/s. The key insight: compute tiles that fit in SRAM, and never write the full attention matrix to slow HBM.

19 TB/s bandwidth
$$ \text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d}}\right) V $$

The Core Insight

Three ideas that changed everything

IO-aware attention isn't one trick — it's three algorithmic ideas working together to keep data in fast SRAM from start to finish.

01

Tiling

Split Q, K, V into blocks that fit in SRAM. Load one tile at a time, compute partial attention, accumulate the result — never touch the full N×N matrix in slow HBM.

02

Online Softmax

Standard softmax needs two passes: one for the max, one for the sum. Flash Attention tracks running statistics and rescales as new tiles arrive, processing the entire row in one streaming pass.

03

Recomputation

During backpropagation, instead of storing the full attention matrix from the forward pass, recompute tiles on-the-fly. Trading a small amount of compute for a large reduction in memory.

How It Works

Online softmax — the algorithmic heart

The key challenge: softmax requires the entire row at once. Flash Attention solves this with a streaming rescaling trick that processes tiles sequentially.

Standard Safe Softmax

$$ m = \max_i x_i \qquad \ell = \sum_i e^{x_i - m} \qquad y_i = \frac{e^{x_i - m}}{\ell} $$

Three passes over the data: find the max, sum the exponentials, then normalize. Each pass reads from HBM — expensive for large N.

Online Softmax (Flash Attention)

$$ \begin{aligned} m^{\text{new}} &= \max(m^{\text{old}}, m^{\text{local}}) \\ \ell^{\text{new}} &= e^{m^{\text{old}} - m^{\text{new}}} \ell^{\text{old}} + e^{m^{\text{local}} - m^{\text{new}}} \ell^{\text{local}} \end{aligned} $$

One pass. Maintain running max m and sum ℓ. When a new block arrives, rescale the previous result if the new block has a larger max — mathematically identical to the three-pass version.

The Tiling Loop

$$ \text{For each tile } (i, j):\quad S_{ij}^{\text{local}} = Q_i K_j^T,\quad \tilde{P}_{ij} = e^{S_{ij}^{\text{local}} - m_i},\quad O_i \leftarrow \text{rescale and accumulate} $$

The outer loops iterate over Q tiles and K/V tiles. For each pair, compute the local attention scores entirely in SRAM, update the online softmax state, and accumulate into the output — without ever writing the full S matrix to HBM.
